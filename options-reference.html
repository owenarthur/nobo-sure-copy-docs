Options reference

Everything you can set when starting a copy, what the default is, and exactly what each choice does. In the current version, every copy goes from an existing space you can view into a brand-new space the app creates — the form fields and options below are the complete surface.

The form

Source space

The space to copy. The dropdown is searchable and lists only spaces you can view — the app checks your own permissions, not its own, so you can't copy content you couldn't otherwise see at the space level.

New space name

The display name for the destination space. Required. The app creates the space with this name during the *Preparing* phase.

New space key

The key for the destination space. Rules:

  • Must start with a letter; letters and numbers only. The field flags anything else as invalid, and the same rule is enforced again when you press Start copy ("Space key must start with a letter and contain only letters and numbers.").
  • Auto-suggested from the name as you type: letters and digits are kept, everything else dropped, uppercased, trimmed to 10 characters — "Marketing Handbook 2026" suggests MARKETINGH. If the name starts with a digit, an S is prefixed. Once you edit the key by hand, it stops auto-updating.
  • Must be different from the source space's key, and not already in use on your site. Both are checked before the job starts; you'll get a plain message either way (e.g. A space with key "MKT2" already exists. Choose a different key.).

Options

| Option | Default | What it does |

|---|---|---|

| Copy attachments | On | For every page, downloads each attachment from the source and uploads it to the page's copy. References to attachments — including images on one page that display a file attached to another — resolve correctly in the copy because page references are rewritten (see How copying works). Turn it off for a structure-and-text-only copy; the copy's attachment counter will read 0 and any attachment references in page bodies will still point at files in the source space. |

| Copy labels | On | Re-applies every label on every source page to its copy, preserving the label's prefix. A failure to copy one page's labels is recorded as a page-level issue and the job continues. |

| Source homepage handling | Merge | See below — this is the one option worth thirty seconds of thought. |

Source homepage handling: merge vs nest

Confluence auto-creates a homepage in every new space, so the source homepage needs a decision:

  • Merge source homepage content into the destination homepage *(default, recommended)* — the source homepage's content (links rewritten) is written into the auto-created destination homepage, which keeps its own title. No extra page is created; anything that linked to the source homepage now links to the destination homepage. One space, one homepage.
  • Copy source homepage as a child page of the destination homepage — the source homepage becomes its own page, with its original title, nested directly under the new space's homepage. Choose this when the source homepage is a genuine content page you want preserved as-is, or when the destination homepage should stay a clean landing page.

Either way, links *to* the homepage are remapped correctly — merge maps them to the destination homepage, nest maps them to the copied page.

Per-option interactions and fine print

  • Options are fixed at start. They're stored with the job when you press Start copy; you can't change them mid-copy. If a failed job is retried, it retries with its original options.
  • Attachments and labels count toward the completion summary ("151 pages copied, 96 attachments, 210 labels") only when their option is on.
  • What you can't choose (yet): copying into an *existing* space or under a parent page, copying a single page tree rather than a whole space, copying footer comments, and copying blog posts are not options in the current version. See Limitations for the full honest list and the roadmap note.

What happens when you press Start copy

In order, before any content is touched:

1. Field validation (source chosen, name present, key valid, key ≠ source key).

2. Permission check, as you: the app verifies that *your* account can view the source space. If not: You don't have permission to view space "KEY" (or it doesn't exist). Ask a space admin for access, then try again.

3. Destination key check: the key must not belong to an existing space.

4. The job is created and queued. The view switches to live progress, and the job appears in Recent copies (the history keeps your 20 most recent jobs).
