Options reference

Everything you can set when starting a copy, what the default is, and exactly what each choice does. Sure Copy copies in three modes — a whole space into a new space, a page tree into an existing space, or a single page — and the options below apply across them (a few are mode-specific, noted as such).

Copy modes

Pick what to copy and where it lands:

  • Whole space → new space. Copies an entire source space into a brand-new space the app creates (name + key you choose). This carries the most fidelity: pages, hierarchy, attachments, labels, folders, page properties, blog posts (optional), the space description, and the source's page templates.
  • Page tree → existing space. Copies a page and its descendants into a space that already exists, placed under a parent page you choose in that space. The destination space keeps its own identity (its description, templates and existing content are untouched).
  • Single page. Copies one page, with or without its descendants (a "copy descendants" toggle). Lands under a parent you choose in the destination space.
  • "Copy page tree to…" action. Every page's ••• menu has a Copy page tree to… action that opens the copy dialog pre-scoped to that page and its descendants — the same engine, launched from the page itself instead of the app's main screen.

Link rewriting, resume-safety, the completion report, and the honest "not copied" inventory work identically in every mode; only the copied set and the destination differ.

The form

Source

Whole-space mode: the space to copy, chosen from a searchable dropdown that lists only spaces you can view — the app checks your permissions, not its own, so you can't copy content you couldn't otherwise see. Page-tree / single-page mode: the source is the page you launched from (via the ••• action) or the page you pick.

Destination

  • New space (whole-space mode): you give a new space name (required — used to create the space in the Preparing phase) and a new space key.
  • Existing space (page-tree / single-page mode): you choose the destination space and a parent page under which the copied tree is placed.

New space key (new-space mode)

  • Must start with a letter; letters and numbers only. The field flags anything else as invalid, and the same rule is enforced again when you press Start copy ("Space key must start with a letter and contain only letters and numbers.").
  • Auto-suggested from the name as you type: letters and digits are kept, everything else dropped, uppercased, trimmed to 10 characters — "Marketing Handbook 2026" suggests MARKETINGH. If the name starts with a digit, an S is prefixed. Once you edit the key by hand, it stops auto-updating.
  • Must be different from the source space's key, and not already in use on your site. Both are checked before the job starts; you'll get a plain message either way (e.g. A space with key "MKT2" already exists. Choose a different key.).

Options

OptionDefaultWhat it does
Copy attachmentsOnFor every page, downloads each attachment from the source and uploads it to the page's copy. References to attachments — including images on one page that display a file attached to another — resolve correctly in the copy because page references are rewritten (see How copying works). Turn it off for a structure-and-text-only copy; the copy's attachment counter will read 0 and any attachment references in page bodies will still point at files in the source space.
Copy labelsOnRe-applies every label on every source page to its copy, preserving the label's prefix. A failure to copy one page's labels is recorded as a page-level issue and the job continues.
Copy descendants (single-page mode)OnWhether a single-page copy also brings the page's child pages. Off copies just the one page.
Copy blog posts (whole-space mode)OnRecreates the source space's blog posts — body, attachments, labels, and in-body links (rewritten like page links) — in the new space. Blog posts belong to a space rather than a page tree, so this applies to whole-space copies only. Off leaves them behind (and the report inventories them as "not copied"). A blog post's original date and author are not preserved — each copy is created fresh.
Copy footer commentsOffRecreates each page's footer (page) comments on its copy, with links inside comment bodies rewritten and threaded replies preserved where the API allows. Each copied comment is prefixed with an attribution line naming the original author (the API authors every created comment as the app). Inline comments are never copied (they can't be re-anchored). Off means no comment API call is made at all.
Recreate page restrictionsOffReapplies each source page's view/edit restrictions to its copy (same users and groups — this is a same-site copy). This is the one option that touches access control and the only one needing a broader scope (write:content.restriction), so it stays off by default; off means no restriction API call is made. Any principal it can't reproduce is reported on the job, never silently dropped. See Limitations and the security page.
Source homepage handling (new-space mode)MergeSee below — the one option worth thirty seconds of thought.
Rename on copyOffOptionally transforms copied page titles with a prefix, a suffix, and/or a find-and-replace, so the copies don't collide by name with the originals (useful when copying into the same space or an existing space). Link rewriting still tracks the renamed pages, so internal links resolve to the renamed copies.

Source homepage handling: merge vs nest (new-space mode)

Confluence auto-creates a homepage in every new space, so the source homepage needs a decision:

  • Merge source homepage content into the destination homepage (default, recommended) — the source homepage's content (links rewritten) is written into the auto-created destination homepage, which keeps its own title. No extra page is created; anything that linked to the source homepage now links to the destination homepage. One space, one homepage.
  • Copy source homepage as a child page of the destination homepage — the source homepage becomes its own page, with its original title, nested directly under the new space's homepage. Choose this when the source homepage is a genuine content page you want preserved as-is, or when the destination homepage should stay a clean landing page.

Either way, links to the homepage are remapped correctly — merge maps them to the destination homepage, nest maps them to the copied page.

Preview, templates, bulk, and scheduling

Beyond a single copy, Sure Copy adds a few power features (each documented in-app):

  • Dry-run preview. Before committing, you can run a read-only preview that reports the plan — how many pages, attachments, folders, blog posts, and which content types will be left uncopied — without writing anything.
  • Templates. Save a copy configuration (source + options, including rename rules with {{variables}}) as a reusable template, then instantiate it later — handy for "clone the Handbook space every time we onboard a client."
  • Bulk copy. Queue several copies as one batch; the app runs them under a concurrency cap and reports each item.
  • Scheduled copies. Set a copy to run on a cadence; a single platform-native hourly trigger fires whichever schedules are due. Zero egress — no external cron.

Per-option interactions and fine print

  • Options are fixed at start. They're stored with the job when you press Start copy; you can't change them mid-copy. If a failed job is retried, it retries with its original options.
  • Counts follow the options. Attachments, labels, folders, blog posts, footer comments and page restrictions each count toward the completion summary only when their option is on; whatever is left uncopied is inventoried in the report.
  • The default set is right for most copies: attachments, labels, folders, page properties and (whole-space) blog posts on; comments and restrictions off. See Limitations for the full, honest list of what is and isn't copied, and why.

What happens when you press Start copy

In order, before any content is touched:

  1. Field validation (source chosen; for a new space: name present, key valid, key ≠ source key; for an existing-space copy: destination space and parent page chosen).
  2. Permission check, as you: the app verifies that your account can view the source. If not: You don't have permission to view space "KEY" (or it doesn't exist). Ask a space admin for access, then try again. When "Recreate page restrictions" is on, it also verifies you hold the right to set restrictions on the source.
  3. Destination check: for a new space, the key must not belong to an existing space; for an existing-space copy, the destination and parent must be reachable.
  4. The job is created and queued. The view switches to live progress, and the job appears in Recent copies (the history keeps your 20 most recent jobs).
