Getting started

Sure Copy copies Confluence content — pages, hierarchy, attachments, labels, folders, page properties and more — and rewrites the internal links so the copy points at itself rather than back at the original. It copies in three modes: a whole space into a brand-new space, a page tree into an existing space (under a parent page you choose), or a single page (with or without its descendants). You can also start a copy from the "Copy page tree to…" action on any page's ••• menu.

This page gets you from install to your first finished copy using the whole-space mode. It takes about two minutes plus however long the copy itself runs. See the Options reference for the other modes and every option.

Install

  1. Find Sure Copy on the Atlassian Marketplace and choose Install (you need to be a site admin to install apps, or ask your admin to).
  2. During installation, Confluence shows the permissions the app requests. Sure Copy asks only for what copying needs: read and write access to spaces, pages, attachments, labels, folders and page templates, a read-only scope to inventory content types it can't copy, plus the comment and page-restriction scopes used only by their optional (off-by-default) toggles, and its own app storage for job records. It requests no admin scopes and sends nothing outside Atlassian — see the FAQ for the full scope list with justifications.

What you need before you start

  • Read access to the source space. The app checks, before anything runs, that you can view the space you're copying. If you can't see it, the copy won't start.
  • A destination. A whole-space copy goes into a brand-new space the app creates — you'll need a free space key (if the one you pick is already taken, you'll be told before anything starts). A page-tree or single-page copy goes into an existing space under a parent page you choose.

Your first copy

  1. Open the app. In Confluence, go to Apps → Sure Copy.
  2. Pick the source space. The dropdown lists only spaces you can view. It's searchable — start typing the space name.
  3. Name the new space. Type a name (e.g. "Marketing Handbook (Copy)"). A space key is suggested automatically from the name — edit it if you want a specific key. Keys must start with a letter and contain only letters and numbers.
  4. Choose your options. Attachments and labels are copied by default; homepage handling defaults to merging the source homepage into the new space's homepage. See the Options reference — the defaults are right for most copies.
  5. Start the copy. Click Start copy. The view switches to live progress.

Screenshot: the copy form with a source space selected and options visible.

Watching it run

The progress view updates every few seconds and walks through three phases:

  1. Preparing — reading the source space and creating the destination.
  2. Creating pages — every page is created in the new space, in hierarchy order, with attachments and labels. You'll see "Creating pages: 47 of 151".
  3. Rewriting links — every copied page's body is written with internal links re-pointed at the copies. You'll see "Rewriting links: 12 of 151".

You can leave the page at any time — the copy runs in the background on Atlassian's infrastructure, not in your browser tab. The Back (copy keeps running) button says exactly what it does; the job also appears under Recent copies, where View brings the progress screen back.

Screenshot: progress view mid-copy showing the phase lozenge and progress bar.

When it finishes

You'll see a completion summary — for example, "151 pages copied, 96 attachments, 210 labels" — with a link to Open the new space. If any individual pages hit problems, they're listed in a panel on the same screen; one problem page never stops the rest of the copy (see Troubleshooting).

Worth a minute after your first copy: open a few pages in the new space and hover over their internal links — they point at pages inside the new space, which is the whole point. What the app doesn't copy (inline comments, whiteboards, and a few other things) is listed honestly in Limitations.
