FAQ

How is this different from Confluence's built-in copy?

Confluence Cloud's native copy works page-by-page (with an option to include child pages). It's fine for small jobs, but it has no space-level copy at all — there is no native "copy this space" — and the copies it makes keep their internal links pointing at the original pages, which is exactly the problem for a self-contained duplicate. It also gives you no itemised outcome report and no resumable background job for large trees.

Sure Copy copies an entire space in one operation — or a page tree into an existing space, or a single page — rewrites every internal link to point at the copies, includes attachments, labels, folders and page properties, shows live progress, and survives interruptions without duplicating pages. The full mechanics are in How copying works.

How is this different from Copy Space or Copy Page Tree?

Those are established Connect-era apps facing Atlassian's Connect end of support; Sure Copy is built on Forge, Atlassian's current platform, which among other things means your content never leaves Atlassian's infrastructure. We've written factual, feature-by-feature comparisons — including what those apps do that we don't — in our migration guides:

Does my content leave Atlassian?

No. Sure Copy is Forge-native: it runs on Atlassian's own infrastructure, calls only Confluence's APIs, and makes no calls to any external service — no third-party servers, no external analytics. Zero egress. Job records (see below) live in Forge storage on Atlassian infrastructure and inherit your site's data residency.

What data does the app store?

Job metadata only: which space was copied to which, the options chosen, progress state, the source-page-to-copy ID map, page titles, per-page error strings, and the account ID of the person who started the copy. Page content passes through the copy operation and is not retained beyond it. The app keeps your 20 most recent job records; older ones are deleted automatically.

What scopes does the app request, and why?

ScopeWhy
Read/write spacesList spaces you can view; create the destination space
Read/write pagesRead source pages; create and rewrite their copies (blog posts share this scope — no extra one is requested)
Read/write attachments (incl. the classic write:confluence-file scope)Download source attachments and upload them to the copies — attachment upload still requires the classic Confluence scope; there is no newer equivalent
Read/write labelsCopy each page's labels
Read/write foldersRecreate the source's folders and nesting so copied pages land under the right folder
Read/write page templatesCopy the source space's user-created page templates into a new space
Read content details (read-only)Inventory the content types the app can't copy (whiteboards, databases) so the report can itemise them — no content is moved by this scope
Read/write commentsUsed only by the optional, off-by-default "Copy footer comments" toggle
Write content restrictionUsed only by the optional, off-by-default "Recreate page restrictions" toggle — the one scope that touches access control
App storageJob records and progress state

Notably absent: no admin scopes, no user management, no space-permission management. Reading a page's restrictions needs no scope; only applying them (the optional toggle) uses write:content.restriction. This small footprint is deliberate — see Limitations and the security page for each scope's justification.

Who can use the app, and whose permissions apply?

Anyone who can open the app can start a copy — but only of a space they can view: the app checks your permission on the source space, as you, before the job is created. The destination is a new space created with your site's default permissions.

How does pricing work? Is it really free for small teams?

Sure Copy is Paid via Atlassian with standard Marketplace per-user pricing — billing, invoicing, and the price ramp are all handled by Atlassian alongside your other Atlassian bills. As with Marketplace-standard pricing, sites with 10 or fewer users pay nothing. The current price per user is on the Marketplace listing, which is always authoritative.

Can I copy between two Confluence sites?

Not currently. Sure Copy works within one site; cross-site copying is a different security model (it would require content to cross the site boundary, which this app never does). See Limitations.

Can I copy a single page tree instead of a whole space?

Yes. Alongside whole-space copies, Sure Copy copies a page tree into an existing space (under a parent page you choose) and a single page (with or without its descendants). Every page's ••• menu also has a "Copy page tree to…" action that copies that page and its descendants into a destination you pick. See the Options reference.

Can I schedule recurring copies, or use a space as a template?

Yes to both. You can save a copy configuration (source + options, with {{variables}} in rename rules) as a reusable template and instantiate it later, and you can set copies to run on a schedule (a platform-native hourly trigger fires whichever schedules are due — no external cron, zero egress). There's also a bulk copy batch mode and a read-only dry-run preview. See the Options reference.

Can I cancel a running copy?

Not in the current version — a running job runs to completion (or failure). Since every copy targets a brand-new space, an unwanted copy is cleanly disposable: let it finish, then delete the destination space from Confluence's space settings.

What happens to the app's data when I uninstall?

The app's stored data — job records and state, as described above — is deleted in line with our privacy policy and Atlassian's Forge platform data-deletion behaviour for uninstalled apps. The spaces and pages the app created are ordinary Confluence content and remain exactly where they are; uninstalling never touches your content.
